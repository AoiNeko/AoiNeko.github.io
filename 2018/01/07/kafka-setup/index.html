    docker启动kafka本地服务

    1.拉取kafka镜像
    docker pull wurstmeister/kafka

    2.配置docker-compose
    version: ‘3’
    services:
    zookeeper:
    image: zookeeper
    ports:

        - "2181:2181"
kafka:
    image: wurstmeister/kafka
    ports:
    - "9092"
    deploy:
      replicas: 2
    environment:
      KAFKA_ADVERTISED_HOST_NAME: 10.35.22.82
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock

    3.启动服务
    docker-compsose up -d

    4.测试kafka
    start-kafka.sh
    docker run –rm -v /var/run/docker.sock:/var/run/docker.sock -e HOST_IP= -e ZK= -i -t wurstmeister/kafka /bin/bash

    sh start-kafka.sh host_ip zookeeper:2181

    创建topic

    bash-4.3# ./kafka-topics.sh –create –topic testtopic –partitions 2 –zookeeper 10.35.22.82 –replication-factor 2
    Created topic “testtopic”.

    查看topic

    bash-4.3# ./kafka-topics.sh –describe –topic testtopic –zookeeper 10.35.22.82
    Topic:testtopic PartitionCount:2 ReplicationFactor:2 Configs:
    Topic: testtopic Partition: 0 Leader: 1005 Replicas: 1005,1004 Isr: 1005,1004
    Topic: testtopic Partition: 1 Leader: 1004 Replicas: 1004,1005 Isr: 1004,1005

    使用console producer和comsumer测试发送与接收

    bash-4.3# ./kafka-console-producer.sh –broker-list 10.35.22.82:32782 –topic testtopic

    my test message
    test message 2
    ^Cbash-4.3# ./kafka-console-consumer.sh –bootstrap-server 10.35.22.82:32782；10.35.22.82:32781 –from-beginning –topic testtopic
    test message 2，，
    my test message
